## 1. 제품 개요

### 1.1 한 줄 요약

여러 심리검사를 웹에서 실시하고, T점수·규준·해석을 제공하며, 사용자·관리자용 PDF 리포트와 관리자 대시보드를 제공하는 **심리검사 분석 서비스**.

### 1.2 목표

- 검사 응답부터 결과·해석까지 **일관된 흐름**으로 완료할 수 있을 것.
- 관리자는 **완료 건수·신뢰도·접속 로그·응답 목록·기준/해석 자료**를 한 곳에서 확인할 것.
- UI와 PDF·API 응답의 **언어(ko/en)** 가 사용자 선택과 일치할 것.

### 1.3 기술 전제 (참고)

- 백엔드: Java 17, Spring Boot, PostgreSQL, Flyway.
- 프론트: Vue 3, TypeScript, Vite.
- PDF: Apache PDFBox.

### 1.4 시각 디자인 (Figma 정렬)

- 제품 UI는 Figma(`Page 3` 등)에서 정의된 **브랜드 네이비·쿨 그레이 배경·Manrope/Public Sans** 톤을 따른다.
- 구체 토큰·컴포넌트 매핑은 `docs/ui-spec.md`를 기준으로 하며, 구현은 `frontend/src/style.css`의 CSS 변수로 관리한다.

## 4. 기능 요구사항

### 4.1 응시자 흐름

1. **검사 목록/소개**: 현재 제공되는 검사를 선택할 수 있다.
2. **검사 진행**: 문항·선택지 표시, 응답 제출.
3. **결과**:
   - 요약(총점·T점수 등 검사 유형에 맞는 지표)
   - 척도별 점수·해석(백엔드 채점·해석 로직 기준)
   - 가능한 경우 차트(T점수 등) 표시
4. **결과 PDF**: 현재 UI 언어와 동일한 locale로 PDF 생성·다운로드 가능해야 한다. (`lang` 쿼리·`Accept-Language` 정책은 백엔드 구현 기준을 따른다.)

### 4.2 다국어 (ko/en)

1. UI에서 **한국어 / English** 전환이 가능하다.
2. 프론트는 선택 언어에 맞게 `Accept-Language`를 설정한다.
3. 백엔드는 검사명, 척도명, 결과 해석, 관리자용 기준/해석 가이드, 선택지 라벨 등을 locale에 맞게 내려준다.
4. **문항 본문** 영어 제공 범위는 README에 명시된 검사 목록과 동일하며, 그 외는 국문 원문 fallback을 허용한다.

### 4.3 관리자

1. **인증**: 관리자 로그인 후 API 접근(예: `X-Admin-Token` 등 구현된 방식).
2. **대시보드**:
   - 검사 완료 수, 결과 수, 신뢰도(α) 등 집계
   - 통계 그래프
   - 접속 로그
   - 응답 목록(페이지네이션)
3. **검사별 참고 자료**: 기준점수(규준), 해석 기준 조회.
4. **PDF**:
   - 대시보드 요약 PDF(요약·그래프 캡처 등 구현된 구성)
   - 기준·해석 PDF(규준 테이블·해석 문단)
   - 개별 응답 결과 PDF(권한·경로는 구현 기준)

### 4.4 API·데이터

- REST API로 검사 메타, 문항, 세션 생성, 제출, 결과 조회, PDF 바이너리 제공.
- DB 스키마·시드는 Flyway 마이그레이션으로 관리.

## 5. 비기능 요구사항

| 항목 | 기대 |
|------|------|
| **보안** | 관리자 기능은 인증 없이 공개되지 않는다. 민감 설정은 서버 설정·환경 변수로 관리한다. |
| **성능** | 일반적인 응시·결과 조회는 실용적인 응답 시간을 목표로 한다. (구체 SLA는 운영 환경에서 별도 정의) |
| **PDF** | 한글·영문 렌더링에 필요한 폰트·인코딩이 깨지지 않아야 한다. |
| **호환** | 최신 브라우저 데스크톱·모바일에서 주요 흐름이 동작해야 한다. |
