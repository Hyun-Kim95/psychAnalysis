## 3. 반경·컴포넌트

- 버튼·작은 컨트롤: **8px**
- 패널·차트 카드·해석 카드: **12px**
- 문항 래퍼: **24px**
- 검사 라이브러리 카드: **32px** (모바일에서는 과도할 수 있어 `ui-spec`에서 브레이크포인트 완화)

---

## 4. 앱 코드와의 갭 (매칭 전)

- `style.css`에 **oklch 기반 shadcn류 변수**와 **Tailwind식 하드코딩 색**이 혼재.
- `body`는 `system-ui` + `#f3f4f6`로 Figma 배경 `#f7f9fb`와 불일치.
- 프라이머리가 `#2563eb` 계열로 Figma `#00288e`와 **색상 체계가 다름**.
- `--heading`, `--text-h` 등이 `:root`에 정의되지 않아 일부 규칙이 무효에 가까움.

---

## 5. 결론

Figma **Home / Assessment / Result** 프레임에서 공통으로 쓰이는 **네이비 브랜드(`#00288e`)·쿨 그레이 배경·Manrope/Public Sans**를 CSS 변수로 고정하고, 기존 컴포넌트 클래스가 이 변수를 참조하도록 리팩터링하는 것이 1차 매칭에 적합하다. 차트 등 데이터 시각화는 동일 톤으로만 조정하고 레이아웃 로직은 변경하지 않는다.
